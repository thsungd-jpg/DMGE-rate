Overview

The DMGE Rate System invoice is a dark-mode, 8-bit/military-styled HTML document rendered server-side and delivered to clients as a PDF or hosted link. The design is defined entirely in self-contained HTML + CSS — no external frameworks or build steps required.

Architecture

Your app populates a data object, injects it into the HTML template via string interpolation or a templating engine, then renders to PDF using a headless browser (Puppeteer / Playwright) or an HTML-to-PDF service.

01
Collect Invoice Data
Assemble the invoice object from your database — sender info, client info, line items, dates, and tax rate.
02
Inject Into Template
Use your templating engine (Handlebars, Jinja2, EJS, etc.) or plain string replacement to populate all {{ variable }} slots in the HTML template.
03
Render to PDF
Pass the fully-populated HTML to Puppeteer or a PDF API. Use A4 or Letter page size at 1x device scale ratio.
04
Deliver to Client
Email the PDF as an attachment, or store in cloud storage and send the client a signed download link.
Font Setup

The invoice uses two Google Fonts. Include both in the <head> of your template. For PDF rendering, embed the fonts locally to avoid network dependency in headless environments.

Color Tokens

All colors are defined as CSS custom properties on :root. Never use raw hex values inline — always reference the token to maintain visual consistency across every element.

CSS :root { /* Brand primaries */ --orange: #FFB347; /* Primary orange — logo, INVOICE, amounts */ --orange-dim: #C47D1A; /* Dimmed orange — borders, payment keys */ --orange-dark: #7A4A00; /* Shadow offset on pixel text */ --pink: #FF3D8C; /* Accent pink — status badge, arrows, sub-labels */ /* Backgrounds */ --black: #080808; /* Page background */ --panel: #111111; /* Page footer background */ --panel-mid: #1A1A1A; /* Hover states */ --panel-light: #222222; /* Table header, TOTAL DUE block */ /* Borders */ --border: #2E2E2E; /* Default dividers */ --border-hi: #3A3A3A; /* Table header borders */ /* Text */ --text-main: #E8D8B0; /* Primary text — names, values */ --text-dim: #7A6A50; /* Secondary text — addresses, notes */ --text-muted: #4A3E2E; /* Placeholder / ghost text */ }

Invoice Data Schema

Structure your invoice payload as follows before passing it to the template engine. All monetary values should be stored as numbers (cents or float) and formatted to 2 decimal places before injection.

JSON { // ── Invoice Meta ────────────────────────────── "invoice_number": "INV-0001", // e.g. "INV-0042" "issue_date": "MM / DD / YYYY", // formatted string "due_date": "MM / DD / YYYY", // formatted string "status": "UNPAID", // UNPAID | PAID | OVERDUE | DRAFT // ── Sender (always DMGE) ────────────────────── "sender": { "name": "DMGE RATE SYSTEM", "address": "Your Street Address", "city": "City, State ZIP", "email": "hello@dmgeratesystem.com", "phone": "+1 (000) 000-0000" }, // ── Client ──────────────────────────────────── "client": { "name": "CLIENT NAME / CO.", "address": "Client Street Address", "city": "City, State ZIP", "email": "client@email.com", "phone": "+1 (000) 000-0000" }, // ── Line Items ──────────────────────────────── "items": [ { "description": "Service / Project Name", "note": "Brief description of deliverable", // optional "qty": 1, "rate": 500.00, "amount": 500.00 // qty × rate, computed server-side } ], // ── Totals ──────────────────────────────────── "subtotal": 500.00, "discount": 0.00, // pass 0 if none "tax_rate": 0, // percentage integer e.g. 8 "tax_amount": 0.00, "total_due": 500.00, // ── Footer ──────────────────────────────────── "notes": "Thank you for your business...", "paypal_email": "pay@dmgeratesystem.com" }
Line Items

Field Reference

FieldTypeRequiredNotes
description string REQ Rendered at 15px --text-main. Keep under 60 chars for single-line display.
note string OPT Sub-line rendered at 12px --text-muted. Prefixed with // automatically.
qty number REQ Integer or decimal. Displayed as-is, right-aligned.
rate number REQ Formatted as $X,XXX.XX. Compute before injection.
amount number REQ = qty × rate. Rendered in --orange at 15px.

HTML Structure Per Row

HTML <tr> <td> <div class="item-desc">{{ item.description }}</div> <div class="item-note">// {{ item.note }}</div> </td> <td class="qty">{{ item.qty }}</td> <td class="rate">${{ item.rate }}</td> <td class="amount">${{ item.amount }}</td> </tr>
ℹ INFO Omit the .item-note div entirely if no note is provided for that item — do not render an empty row. The // prefix is hardcoded in the HTML, not stored in the data.
Totals Logic

Always compute totals server-side. Never rely on client-side JS for financial calculations.

JS // Server-side calculation (Node.js example) const subtotal = items.reduce((sum, i) => sum + (i.qty * i.rate), 0); const discount = invoice.discount ?? 0; const taxable = subtotal - discount; const tax_amount = taxable * (invoice.tax_rate / 100); const total_due = taxable + tax_amount; // Format for template injection const fmt = (n) => n.toLocaleString('en-US', { minimumFractionDigits: 2, maximumFractionDigits: 2 }); invoice.subtotal_fmt = fmt(subtotal); invoice.discount_fmt = fmt(discount); invoice.tax_amount_fmt = fmt(tax_amount); invoice.total_due_fmt = fmt(total_due);

Tax Row Display

Render the tax row label as TAX ({{ tax_rate }}%). If tax_rate is 0, still render the row — display TAX (0%) and $0.00. Do not hide it.

Status States

The status badge appears in the top-right invoice meta block. Four states are supported. Apply the CSS class and color as shown below.

UNPAID
Default state. New invoice awaiting payment.
PAID
Payment received and confirmed.
OVERDUE
Due date has passed with no payment.
DRAFT
Not yet sent. Internal use only.

CSS Per Status

CSS /* UNPAID */ .status-badge { color: #FF3D8C; border-color: #FF3D8C; } /* PAID */ .status-badge { color: #39FF8A; border-color: #39FF8A; } /* OVERDUE */ .status-badge { color: #FF6B3D; border-color: #FF6B3D; } /* DRAFT */ .status-badge { color: #7A6A50; border-color: #3A3A3A; }
HTML Generation

Inject data into the template using your stack's templating engine. Below are examples for common stacks.

Node.js + Handlebars

JS import Handlebars from 'handlebars'; import fs from 'fs'; const source = fs.readFileSync('dmge-invoice-template.html', 'utf8'); const template = Handlebars.compile(source); const html = template(invoiceData);

Python + Jinja2

PYTHON from jinja2 import Environment, FileSystemLoader env = Environment(loader=FileSystemLoader('.')) template = env.get_template('dmge-invoice-template.html') html = template.render(invoice=invoice_data)

PHP

PHP // Using ob_start to capture template output extract($invoiceData); ob_start(); include 'dmge-invoice-template.php'; $html = ob_get_clean();
PDF Export

Use Puppeteer (Node.js) or Playwright to render the invoice HTML to PDF. The settings below ensure pixel-accurate output matching the screen design.

Puppeteer (Recommended)

JS import puppeteer from 'puppeteer'; async function renderInvoicePDF(html) { const browser = await puppeteer.launch({ args: ['--no-sandbox'] }); const page = await browser.newPage(); await page.setContent(html, { waitUntil: 'networkidle0' }); const pdf = await page.pdf({ format: 'Letter', // or 'A4' printBackground: true, // REQUIRED — renders dark background margin: { top: '0', right: '0', bottom: '0', left: '0' }, deviceScaleFactor: 2 // Retina quality }); await browser.close(); return pdf; }
⚠ CRITICAL printBackground: true is mandatory. Without it, Puppeteer strips all background colors and images — the entire dark theme will be lost and the invoice will render as a blank white page.

Playwright Alternative

JS import { chromium } from 'playwright'; const browser = await chromium.launch(); const page = await browser.newPage(); await page.setContent(html, { waitUntil: 'networkidle' }); const pdf = await page.pdf({ printBackground: true, format: 'Letter' }); await browser.close();

Third-Party PDF APIs

If deploying serverless (Vercel, Lambda), use a managed HTML-to-PDF API since headless Chrome is too large. Recommended options:

ServiceKey Setting
PDFShiftSet use_print: false, css_media_type: screen
WeasyPrintSelf-hosted Python. Full CSS support. Background-color safe.
Browserless.ioDrop-in Puppeteer replacement. Set printBackground: true.
APIFlashScreenshot mode. Set format PDF + full page.
Email Delivery

Send the rendered PDF as an email attachment. Use a transactional email provider. The subject line and body should match the DMGE brand voice.

JS // Nodemailer example await transporter.sendMail({ from: 'DMGE Rate System <invoices@dmgeratesystem.com>', to: client.email, subject: `INVOICE #${invoice.invoice_number} — DMGE RATE SYSTEM`, text: `Your invoice for $${invoice.total_due_fmt} is attached.`, attachments: [{ filename: `DMGE-${invoice.invoice_number}.pdf`, content: pdfBuffer, contentType: 'application/pdf' }] });
✓ TIP Name the attachment DMGE-INV-XXXX.pdf so clients can easily identify and file it. Store a copy in your database or S3 for audit trails.
